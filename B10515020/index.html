🙂 使用情境說明(包含流程圖)

我們這組在第二次作業中實作了一些跟 memory 有關的 system call，包刮 brk、sbrk、malloc、calloc、valloc、realloc。在這次的作業中，我們利用了前一次作業實作的 system call，實作一個 in-memory 的小型 key-value database，參考了功能類似的知名資料庫 redis，可以提供 set、get、update、delete、list、count 等功能。

使用情境範例

這樣的 database 的優點在於儲存速度快，可以做為網站後端暫存資料的 database，暫時存放登入的使用者的 token 等容許遺失的資料。

🏃 實作過程(修改哪些檔案[含圖片])

最一開始我們想實作的內容與 process 有關，因此我們在程式中加入了 ps、Ctrl+C 中斷 process、suspend & resume process，因此在新增、修改的檔案中有些是與 process 相關的。雖然最後改成實作 database，但這些跟 process 相關的功能有的也蠻實用的，因此我們仍然將這些功能保留下來。

主要跟 database 相關的功能在 memDB、umalloc 這兩個檔案中，包刮 database 的邏輯、記憶體的管理、配置。
另外為了 parse 指令，我們在 ulib 內加入了 strtok，可以把字串照指定的 token 分割成好幾個子字串。

😎 結論

原本要做跟 process 相關的內容，但在過程中碰到很多困難，參考了很多人的做法、trace 了很多 xv6 的 code 後，我們發現仍然做不出來，在 deadline 快到的時候決定換一個比較簡單的主題，做一個 in-memory 的 database。雖然說比較簡單，但因為沒有實做過 database 的經驗，我們沒有選擇複雜的 SQL database，而是選擇 key-value database，也剛好符合 in-memory database 存取快速、方便的特性。

寫作業的過程中，為了研究怎麼實作想達成的功能，我們 trace 了很多 xv6 的 code，在過程中也同時複習了課本中的內容，對於 OS 的運作有深刻的理解，也發現要寫出一個好的 OS 是一件十分不容易的事。
