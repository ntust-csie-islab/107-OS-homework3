🙂 使用情境說明(包含流程圖)

透過 一個user level program “sender” 傳資料到 buffer ，buffer建立在kernel裡，接著可以用另一個user level program “receiver”去讀取以傳送到kernel buffer的資料，可以用之前寫好的scanf()去取出來。資料傳輸的型態我們用array 和pointer 去把字串存到 buffer裡，在用kernel裡面的address去reference pointer然後把資料取出來。透過kernel裡面的buffer使用者可以互相傳訊息，類似像在傳簡訊。用以寫好的producer/consumer 的概念去操作kernel buffer，裡面資料可以把它當作item去看。當buffer是空 (empty, item<0) 的時候，producer (sender)就可以寫，一直到buffer滿(full)為止，但是consumer (receiver)卻不能運作，因為沒有資料可以操作。同理，當buffer是滿 (full, item>0) 的時候，producer (sender)就無法運作、不可以寫，因為沒位子，一直到buffer有至少一個空間為止，但是consumer (receiver)能運作，資料可以去隨意操作，一直到資料完全讀取完後，buffer會回到初始狀態 (empty, item<0)。

把alarmtest 變成可以把中斷前的位子處存在暫存器裡，中斷後再把位子回傳到user level，但是改完後效果沒有達到。本來想說用stack把資料位子存在裡面，之後執行完就可以回傳被中斷前的位子。改的程式碼是在trap.c 和sysproc.c 。利用這種方式把被測試後的程式（被中斷的同一個）去修改它。畢竟已經知道是哪一個被中斷，也知道它目前位子，就可以想辦法去修改或是替換成另一個更有效的程式。把這個修改完的alarmtest 跟新增的time system call去合併，去達到非常好的效果。可以透過 time system call去把執行的程式時間讀取出來，可以準確地算出實際執行的時間 CPU Time。我們有了process的位子和它總執行時間就能想辦法把它CPU Time 減少，可以達到更有效的 CPU。每一個程式都減少一些，可以把全部執行的時間降低，可以把效率提升不少。
在proc.h裡創造cpu_ticks_in 和cpu_ticks_total。可以透過新增的 cpu_ticks_in 和cpu_ticks_total去運算process 消耗的 CPU time 。在process執行前和process執行後cpu_ticks_in 和cpu_ticks_total 的值 都會更新，並且在 allocproc設為0。

CPU Time

為了要偵測和測試CPU Time 正常運作，我們測試shell process 使用的 CPU Time 用crtl-p 印出shell的CPU Time。接著我們連續按 return key 讓 shell執行 null commands這樣可以增加 CPU usage。
現在來測試 user command裡的time command。我們跑程式然後將不同指令去執行，再透過time 去看每一個使用的 CPU Time是多少。
測試 null，因為它是空指針常量，沒有運算所以執行的時間是在0 和1秒之間，很接近 0，也可似為0，所以執行結果沒問題。
測試ls，因為它執行的時間也很快是在0 和1秒之間 是在合理時間完成，所以執行結果沒問題。因為ls執行比較快ls結果會先出來，接這著會有time結果。
測試forktest，因為它執行的時間比較慢2秒左右，所以執行結果沒問題。forktest結果會先出來，接這著會有time結果。
測試echo，因為它執行的時間也很快是在0 和1秒之間 是在合理時間完成，所以執行結果沒問題。因為echo執行比較快echo結果會先出來，接這著會有time結果。
測試time echo (和上一個echo一樣但多了一個 time執行)，因為它執行的時間還是會很快是在0 和1秒之間 是在合理時間完成，所以執行結果沒問題。因為echo執行比較快echo結果會先出來，接這著會有time echo結果 、最後才有time結果。
最後測試隨機不存在的指令 “badcommand”，因為它沒有執行的時間也沒有運算時間，很接近 0，也可似為0，所以執行結果沒問題。
因為CPU 執行時間非常快，都不會超過1秒。我們可以透過time去作出kernel service服務，讓使用者找出花很多CPU Time時間的process。想辦法去改散它，或用比較有效的其他process 代替。

time.c




































































#include "types.h"
#include "user.h"
#include "date.h"
//定義秒數與時數
#define HSECONDS 3600
#define MSECONDS 60

uint
rtcdate_to_sec(const struct rtcdate *r)
{
  return (r->hour * HSECONDS) +
         (r->minute * MSECONDS) +
         (r->second);
}

int
main(int argc, char *argv[]) 
{
  struct rtcdate start, end;
  char *filename, **args;
  int pid;
  uint start_sec, end_sec, elapsed_sec;

  if (argc < 2)
  {
    printf(2, "Usage: %s command [args...]\n", argv[0]);
    exit();
  }

  filename = argv[1];
  args = argv+1;

  if (date(&start)) 
  {
    printf(2, "date failed\n");
    exit();
  }
  pid = fork();
  if (pid > 0 /*Parent*/) 
  {
    wait();
    if (date(&end))
    {
      printf(2, "date failed\n");
      exit();
    }
  } 
  else if (pid == 0 /*Child*/)
  {
    exec(filename, args);
    printf(2, "exec %s failed\n", filename);
    exit();
  }
  else /*Error*/
  {
    printf(2, "fork failed\n");
    exit();
  }

  start_sec = rtcdate_to_sec(&start);
  end_sec = rtcdate_to_sec(&end);
  elapsed_sec = end_sec - start_sec;

  printf(1, "%s ran in %d minutes and %d seconds\n",
            filename, (elapsed_sec/60), (elapsed_sec%60));
  exit();
}

trap.c from alarmtest









































if (proc->alarmtick_cnt == proc->alarmticks) {
   proc->alarmtick_cnt = 0;
   // we have to check whether tf->esp is valid or not because
   // we do write operations of the area tf->esp points to.
   if (tf->esp < KERNBASE) {
     // return address of the interrupt
     *((uint*)(tf->esp - 4))  = tf->eip;

     // save caller-saved register on user stack
     *((uint*)(tf->esp - 8))  = tf->eax;
     *((uint*)(tf->esp - 12))  = tf->ecx;
     *((uint*)(tf->esp - 16)) = tf->edx;

     // use the user stack to do some tricky things
     // install following code on the user stack
     // -------
     // movl $SYS_restore_caller_saved_reg, %eax
     // int $T_SYSCALL
     // ret           ; actually we can ommit the ret instruction.
     // -------
     // the installed code will call a system call which
     // will help restore the caller saved register
     *((uint*)(tf->esp - 20)) = 0xc340cd00;
     *((uint*)(tf->esp - 24)) = 0x000018b8;

     // set the return address of handler to the start
     // of the above code
     *((uint*)(tf->esp - 28)) = tf->esp - 24;

     // set the user stack pointer to return address of the handler
     tf->esp = tf->esp - 28;

     // when trap return, it returns to the
     // alarmhandler
     tf->eip = (uint)proc->alarmhandler;

     proc->tf = tf;
     proc->alarmleft = 1;
   }
 }

sysproc.c from alarmtest

























// the new syscall for restore caller saved registers
int sys_restore_caller_saved_reg(void) {
  struct trapframe *tf = proc->tf;
  if (tf->esp >= KERNBASE) {
    return -1;
  }

  tf->esp = tf->esp + 8;
  tf->edx = *((uint*)tf->esp);

  tf->esp = tf->esp + 4;
  tf->ecx = *((uint*)tf->esp);

  tf->esp = tf->esp + 4;
  tf->eax = *((uint*)tf->esp);

  tf->esp = tf->esp + 4;
  tf->eip = *((uint*)tf->esp);

  tf->esp += 4;

  proc->alarmleft = 0;
  return 0;
}

😎 結論

可以讓使用者體驗傳訊息的概念，在kernel裡面透過buffer能穿輸訊息。兩個不同的使用者可以同時去讀寫同一分資料。因為在kernel裡面priority比user interface高，資料可以保存著。資料傳輸的型態我們用array 和pointer 去把字串存到 buffer裡，在用kernel裡面的address去reference pointer然後把資料取出來。透過kernel裡面的buffer使用者可以互相傳訊息，類似像在傳簡訊。

當我們知道process花的時間，可以想辦法去改散、改良它或者用另一個更有效的process去替換而取代它。可以透過 time system call去把執行的程式時間讀取出來，可以準確地算出實際執行的時間 CPU Time。我們有了process的位子和它總執行時間就能想辦法把它CPU Time 減少，可以達到更有效的 CPU。每一個程式都減少一些，可以把全部執行的時間降低，可以把效率提升不少。
